Design - board4fun Website

Repo

github repo

Overall Target

Build a website that supports multi-player playing board games in an anonymous and real-time fashion. Users can simply select a board game to start or enter a game room link/id to join others.

Requirements

  1. Support board game ‘U Draw I Guess’:
    • Implement real-time draw/display functionality at drawer/guesser side.
    • Implement guessing functionality at backend, check guesser’s answer is correct or not.
    • Implement game rules to add proper VPs to drawer/guesser.
    • Implement feedback system to give credit to best drawer/guesser.
    • Able to leave/rejoin the game seamlessly.
    • Able to add/use customized vocabulary.

Tech Stack

Basic Workflow

  • user input
  • based on input interaction, ajax sends out corresponding request
  • server receives request, reacts correspondingly, broadcasts response (if any)
  • client receives response, ajax partially/fully reload webpage
  • repeat

UI

  • html5 as static webpage language
  • css as style language, bootstrap library is used
  • js as programmatic language for client, multiple libraries are used:
    • jQuery to select html/css elements
    • ajax to send request, and udpate html page based on response

Backend

  • REST API design, based on websocket/http protocol
  • js as programmatic language for server, multiple libraries are used:
    • express to route and serve http request
    • socket.io to establish websocket connection and manage communication
  • mysql/sqlite as database (or use some nosql key/value storage?)
